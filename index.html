Predicting Diabetic Patient Readmission

A Data Exploration and Preparation Project using Electronic Health Records.

Project Overview

This project demonstrates a comprehensive data analysis workflow on a large electronic health record (EHR) dataset of diabetic patient encounters. My task was to explore and prepare the data for a future machine learning model to predict patient readmission.

The project highlights key data skills, including data cleaning, exploratory data analysis (EDA), feature selection, and the ability to derive actionable insights from complex medical data.

Data Preparation and Cleaning

Handling Missing Values

  • The `weight` column was removed due to an excessive number of missing values.
  • Missing values in the `race` column (`?`) were replaced with a `Missing` category to preserve the integrity of the data distribution.
  • The `Unknown/Invalid` values in the `gender` column were replaced with the mode, `Female`, as the count was not large enough to skew the data.

Handling Invalid Features

  • Drug columns like `chlorpropamide`, `acetohexamide`, and `tolbutamide` were removed because they had little to no variability, making them useless for prediction.
  • A high correlation (0.98) was found between the `diabetesMed` and `diabetes` columns. This insight is crucial for careful feature selection to avoid redundancy in the model.
